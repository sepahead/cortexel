## 1. Executive summary & verdict

**What Cortexel actually is (shipped).** A TypeScript library that gives an LLM
agent a *typed, hostile-input-hardened, honesty-enforcing path* from spiking-neural
simulation output to a figure. An agent emits a plain-JSON `VizSpec`; a lenient
envelope gate turns "a JS value" into "literal, bounded, acyclic, accessor-free
JSON" (`cloneExactJson`, `core/vizSpec.ts:197-375`); a strict per-skill gate then
checks params against one of 26 closed zod schemas, checks provenance flags, and
computes a **mandatory honesty caption from flags alone**; a checked renderer draws
it â€” either as a canonical React+SVG chart (19 skills, `react/charts/`) or a
host-injected R3F 3D scene (population voxel hub, neuron cloud, evidence knowledge
graph). Four skills have `scene:null` and route to an *external* renderer
(d3/matplotlib/manim) through a parallel gate. A generated 284 KB
`skills.manifest.json` republishes the whole contract for non-TS hosts. `dist/` is
committed so git-dependency consumers install with no build step. 441 tests pass;
typecheck is clean; the build is deterministic (verified: two fresh builds produce
byte-identical chunk contents).

**The verdict.** The engineering is genuinely strong and unusually disciplined for
a pre-1.0 project. Two things are **real, citable novelty**: (1) *fail-closed
scientific-honesty provenance as a security property* â€” no viz grammar in the
lineage (Vega-Lite, Plotly, ECharts, Gosling, MolViewSpec) derives a tamper-resistant
disclosure from machine-checkable flags and treats a flag that suppresses it as a
bug requiring a test; and (2) *adversarial hardening of the spec envelope* â€” treating
the spec author as hostile (reject `__proto__`, `-0`, sparse arrays, `JSON.rawJSON`,
Proxy length-traps, accessors) is exactly right when the author is an LLM, and
nobody else in this space does it. **The overstated claim** is "grammar": Cortexel
has no encoding channels, no transforms, no composition algebra, no interaction â€” the
four pillars that define a grammar of graphics. It is a *closed catalog of
parameterized scene templates*, an architectural sibling of Plotly's trace schema,
not of Vega-Lite. That is a legitimate and arguably *safer* design for an LLM target
(a bounded enum is easier to validate, route, and repair than an open algebra), but
the README should say "hardened, honesty-enforcing figure catalog for agents," not
"grammar."

**The central premise â€” does it hold?** Partly, and the gap is the most important
finding in this report. Cortexel makes figures *provably well-formed and provably
labelled*, and it makes a specific class of dishonest **rendering** impossible
(interpolating across gaps, painting structural absence as measured zero, bridging
omitted bins). It does **not** make figures *true*: every provenance flag is
self-asserted by the same agent that made the figure, and nothing checks that the
rendered **data** matches the claimed provenance
(`core/skills/provenanceKeys.ts:289-328` verifies declared-string == param-string
equality, nothing more). Honesty is enforced on *form*, asserted on *substance*.
That is a defensible scope â€” but the docs oversell it, and there is at least one
input shape where the form-honesty itself **fails open** (see the headline table).

**One audit concern was refuted on inspection.** The `contract-core` audit worried

lenient path is reachable only via an explicit trust opt-in. Corrected here so it
does not propagate as a false problem.

## 2. What is included â€” a complete inventory

### 2.1 The contract core (`core/vizSpec.ts`, `core/designLaws.ts`, barrels)
- **`SCENE_NAMES`** (frozen tuple) drives both the `SceneName` TS union and the
  runtime zod enum â€” a genuine single source of truth (`designLaws.ts:6`,
  `vizSpec.ts:445`).
- **`cloneExactJson`** (`vizSpec.ts:197-375`, ~178 LOC) is the trust boundary: it
  rejects non-finite numbers, `-0`, cycles, sparse/decorated arrays, accessors,
  symbols, non-plain prototypes, `BigInt`, `undefined`, functions, and
  `toJSON`/`rawJSON` shape games, within depth/node/string budgets
  (`CORTEXEL_JSON_LIMITS`: depth 32, 500k nodes, 100k string, 5M total).
- **`ProvenanceSchema`** pins `calibrated_posterior: z.literal(false)`
  (`vizSpec.ts:406`) â€” the load-bearing honesty invariant.
- Frozen policy constants (`CORTEXEL_JSON_POLICY`, `ENVELOPE_NORMALIZATION_POLICY`)
  are mirrored into the manifest for non-TS hosts.
- **Caveat:** the envelope is *lenient by design* â€” it does not check that `skill`
  is a real `SKILL_ID` or that `skill`â†”`scene` agree (`vizSpec.ts:452`). Strictness
  lives downstream in the skill gate. `designLaws.ts` also leaks ~8 dead
  backdrop-era exports (`SCENE_FRAMING`, `CAMERA_PRESETS`, `NeuralScene*`, â€¦) into
  the public API via `export *`.

### 2.2 The skill axis (`skillIds.ts`, `registry.ts` 2657 LOC, `router.ts`)
- 26 ids in a frozen tuple â†’ a `Record`-typed `SKILL_REGISTRY` (key completeness is
  type-enforced). Each contract carries device family, scene-or-null, required
  params/provenance flags/keys, portable cross-field constraint declarations, and a
  worked example.
- **`router.ts`** is the executable `nest.viz_router`: it maps a raw NEST dict to a
  concrete skill+scene, fail-closed for unknown/ambiguous/`scene:null`. The
  familyâ†’skill index is *derived* from the registry at load with a duplicate-kind
  throw (`router` derivation, not hand-written).
- **Structural risk:** the registry is 2657 hand-written lines; the four matrix
  skills and two degree skills are near-verbatim copy-paste
  (`registry.ts:914-1131`). Constraint-kind names are triplicated (union type +
  kinds array + semantics object) with only ~10 of 35 spot-checked
  (`registry.ts:98-133,236-272,273-451`). `describeSkill` hand-clones the same
  nested shape the freeze loop hand-enumerates â€” two parallel field lists that must
  be edited in lockstep.

### 2.3 Params schemas (`params.ts` 2299 LOC)
One strict `.strict()` zod schema per skill + shared primitives (`gpuNumber`
Float32-clamped, `gpuArray`, `idArray`, `timeArray`, `displayText`) + reusable
`superRefine` helpers (monotonicity, uniform bins, normalized-mass, sparse-matrix,
window-edge, spatial-bounds). These bounds *are* the published contract (re-emitted
into the manifest). Notable internal inconsistencies (all verified): `timeArray`
skips the Float32 clamp (finding 5); `bin_width_ms` has three different bounds
across sibling skills; derived-value tolerance is `Object.is` (bit-exact) for delay
but `approximatelyEqual` for degree; `Spatial3DObjectSchema` uses `.passthrough()`
and `animation_replay` uses `z.unknown()` state â€” the only two escape hatches from
"typos fail closed."

### 2.4 The gates (`paramPreflight.ts` 867, `validateSkillInvocation.ts` 616)
Two-stage preflight (raw pre-clone length fuse â†’ post-clone element fuse) *before*
the zod schema, which is the correct amplification guard (`preflight before parse`).
`calibrated_posterior===true` and version mismatch are read from the raw payload
before `validateVizSpec`. The gate returns the mandatory caption so the renderer
cannot omit it. **Weaknesses:** per-skill field shapes are duplicated across four
sites (`ALLOWED_PARAM_FIELDS` + two preflight switches + registry zod) with no single
source; the raw pre-clone fuse is TOCTOU-defeatable by a Proxy (validation stays
correct, only amplification-avoidance fails); and the three bidi/`toString` leaks
above.

### 2.5 Honesty layer (`provenance.ts`, `safeRuntime.ts`, `provenanceKeys.ts`)
`mandatoryDisclosure(p)` computes a prefix from flags; a caller `caption` is
appended only as a re-escaped, bidi-isolated **"Caller note (unverified)."**
`provenanceParamConstraintError` checks declaredâ†”param *string equality* under
`Object.is` with prototype-safe path walking. **Two honesty caveats verified:** the
header claims the disclosure derives "only from machine-checkable flags," but
`mandatoryDisclosure` also branches on `p.source` (caller free text,
`provenance.ts:90-91`); and nothing verifies the DATA is real â€” only that declared
strings match param strings.

### 2.6 Host-invocation escape hatch (`hostInvocation.ts`, `verify.ts`, `authoring.ts`, `examples.ts`)
`scene:null` skills route to an external renderer through
`validateHostRendererInvocation`, which enforces the *same* params/provenance/version
rigor as the in-process gate (it is **not** a validation bypass) but only *returns*
the caption. `detectEmptyScene` (verify.ts, one ~290-LOC function) re-validates
SceneData shape against a hand-maintained `SCENE_DATA_FIELDS` mirror. `authoring.ts`
is the authorâ†’validateâ†’repair loop with prompt-injection-safe error formatting.

### 2.7 NEST boundary (`adapters.ts`, `shapes.ts`, `safeInput.ts`, `analysis.ts`, `topology.ts`)
- **`safeInput.snapshotNestInput`** is the hostile-input wall (500k-node cap,
  prototype/accessor/cycle/depth checks) before zod re-hardens with descriptor
  snapshots. Accepts numeric typed arrays (unlike the strict VizSpec boundary).
- **`analysis.ts`** (677 LOC): population rate, ISI, PSTH, correlogram â€” half-open
  `[start, stop)` binning with a bounded roundoff snap capped at 1e-7 fractional,
  failing closed to `INDETERMINATE` beyond it. Population rate normalized by
  *declared* sender count (correct). ISI computed within-sender only.
- **`topology.ts`** (1127 LOC): connection graph, adjacency/weight/delay matrices,
  degree/delay distributions, spatial map. Mandatory declared node universe so
  isolates survive; singular/plural SynapseCollection forms accepted but never
  mixed, scalars never broadcast; multapses preserved; structural absence never
  painted as measured zero; out-degree refused for `mpi_target_rank_local` scope.
- **Boundary hardening follow-up â€” `resolved`:** shared single-source/per-series
  timestamps reject duplicates, typed-array lengths use the intrinsic slot rather
  than subclass accessors, and recorder adapters share the deep safe-input snapshot.

### 2.8 Corpus knowledge graph (`corpusKnowledgeGraph.ts` + `KnowledgeGraph3DParamsSchema`)
A no-throw adapter from an Engram `CorpusEntityGraphResponse` to strict params:
epistemics are a frozen `DERIVED_ADVISORY` spread onto every element (never read
from input), and the output is re-validated through the strict schema. The
~150-LOC `superRefine` enforces duplicate-id rejection, endpoint-kind matching,
â‰¤9 parallel edges/pair, and "evidence cannot terminate inside the graph." **Caveat:**
for the anchors the adapter itself emits, that anti-circularity guarantee is
*cosmetic* â€” every node's sole evidence is a `graph_snapshot_record` whose
`record_id` is self-derived from the node's own id and never checked to resolve
(`corpusKnowledgeGraph.ts:359-363`; `params.ts:1723-1731`).

### 3.1 Declarative viz grammars (Cortexel's true lineage)
- **[Vega-Lite: A Grammar of Interactive Graphics](https://vis.csail.mit.edu/pubs/vega-lite/)**
  (Satyanarayan et al., IEEE TVCG/InfoVis 2017) â€” the yardstick. Cortexel is *not* a
  grammar in this sense: no encoding channels, transforms, composition, or
  interaction. Cite it to *contrast*, not to claim kinship.
- **[Vega/Vega-Lite JSON Schema](https://github.com/vega/schema)** â€” versioning is
  done via a resolvable `$schema` URL with major/minor resolution. Cortexel has a
  single exact-literal `specVersion` and no bindable schema URL. Adopt this.
- **[Plotly figure schema](https://plotly.com/python/figure-structure/)** â€” this,
  not Vega-Lite, is Cortexel's architectural sibling: a closed enum of parameterized
  figure types with strict per-field validation. Frame Cortexel relative to Plotly.
- **[MolViewSpec](https://pmc.ncbi.nlm.nih.gov/articles/PMC12230705/)** (Mol*
  extension, 2024/25) â€” the closest existing analog to Cortexel's exact design
  (declarative 3D scientific scene, host-agnostic, multi-renderer, JSON-Schema +
  **Pydantic** dual validation that actually ships). Cortexel's "host SHOULD mirror
  as a Pydantic model" is aspirational; MolViewSpec ships it. **Prior art for the
  pattern â€” must cite.**
- **[Gosling](https://ieeexplore.ieee.org/document/9557192/)** (TVCG 2022) â€” a
  domain-specific scientific grammar that stayed *compositional* while specializing
  to genomics. The right precedent for "should Cortexel have a track/mark layer
  instead of a fixed enum?"
- **[A Declarative Grammar of Flexible Volume Visualization Pipelines](https://pubmed.ncbi.nlm.nih.gov/30130223/)**
  (TVCG/SciVis 2018) â€” a declarative scivis-3D vocabulary Cortexel could borrow
  instead of hardcoding one R3F scene per skill.
- **[Mosaic vgplot](https://idl.uw.edu/mosaic/vgplot/)** (VIS 2024) and
  **[Nebula](https://shuxinhuan.github.io/pub/nebula.pdf)** (TVCG 2021) â€” mark the
  two dimensions Cortexel omits: data-source/lazy-query binding (Cortexel inlines
  bounded literal data, 500k-node ceiling) and multi-panel view coordination.

### 3.2 LLM chart generation & spec-repair loops
- **[VegaChat](https://arxiv.org/abs/2601.15385)** (arXiv 2026) â€” Cortexel's exact

- **[Visualization Generation with LLMs: An Evaluation](https://arxiv.org/pdf/2401.11255)**

- **LIDA** (Dibia, Microsoft), **Data Formulator** (Microsoft Research),

  **Chat2VIS**, **nvBench** â€” **(confirm URL)**. Draco is the interesting one:
  it encodes viz *soundness* as answer-set-programming constraints. Cortexel's
  per-skill `superRefine` invariants are a hand-rolled, domain-specific cousin of
  Draco's constraint approach; worth citing and contrasting (Cortexel constraints
  are imperative and un-unified; Draco's are declarative and solver-checked).

### 3.3 Honesty, deception & mirages in visualization
- **McNutt, Kindlmann & Correll, "Surfacing Visualization Mirages"** (CHI 2020) â€”
  the canonical formal treatment of figures that mislead despite being "valid."
  Cortexel's rendering-honesty invariants (no interpolation across gaps, no
  absence-as-zero, mass-preserving compaction) are concrete anti-mirage rules and
  should cite this. **(confirm URL.)**
- **Draco/VisuaLint "visualization linting"** and **deceptive-viz taxonomies
  (Pandey et al.)** â€” Cortexel is effectively a *render-time linter that fails
  closed*, a stronger enforcement point than post-hoc linting. **(confirm URL.)**
- **Uncertainty visualization (Jessica Hullman; hypothetical outcome plots)** â€” the
  literature Cortexel most conspicuously does *not* engage: it has **no uncertainty
  representation at all** (no error bars, CIs, ensembles, HOPs), yet its whole pitch
  is epistemic honesty. This is the biggest scientific-credibility gap after
  golden-value testing. **(confirm URL.)**

### 3.4 Provenance standards
- **W3C PROV / PROV-JSON / PROV-O**, **RO-Crate**, **nanopublications**,
  **Croissant** (ML dataset metadata), **VisTrails** (provenance in a viz system) â€”
  **(confirm URL)**. Cortexel's provenance object is bespoke and *thin* (a handful
  of self-asserted booleans + a source string), and it does not interoperate with
  any of these. This is a real positioning weakness: a reviewer will ask "why not
  emit PROV, or embed an RO-Crate?" The honest answer is that Cortexel's provenance
  is a *disclosure trigger*, not a *derivation record* â€” but then it should cite
  PROV as the thing it is deliberately **not** trying to be, and consider emitting a
  PROV/RO-Crate sidecar so figures are archivable in standard tooling.

### 3.5 The NEST / computational-neuroscience stack
- **PyNEST** (`nest.Create/Connect`, `GetConnections`, `GetPosition`,
  `SynapseCollection`, multimeter/spike_recorder/weight_recorder/correlation_detector),
  **Elephant** (Electrophysiology Analysis Toolkit) + **Viziphant** (its companion
  plotting lib), **Neo** (the standard neural data-object model), **NEST Desktop**,
  **Insite**, **VIOLA**, **ConnPlotter**, **PyNN**, **NetPyNE**,
  **NeuroML / SONATA** â€” **(confirm URL)**. This is the decisive context:
  **a NEST user already has Elephant+Viziphant+matplotlib in Python.** Cortexel's
  analysis transforms (ISI, PSTH, rate, correlogram) *reimplement functions Elephant
  already provides and validates.* That is either (a) a reason to depend on/interop
  with Elephant rather than reimplement, or (b) a reason to add **golden-value tests
  against Elephant output** so the reimplementation is provably equivalent. Right
  now it is neither, and finding 7 (no golden tests) makes the reimplementation an
  unvalidated liability in a field that will not forgive a wrong PSTH.
- NEST semantics that are easy to get wrong and that Cortexel handles **correctly**:
  rank-local `GetConnections` under MPI (out-degree refused), declared node universe
  so isolates survive, within-sender ISI ordering, structural-absence preservation.
  Credit where due â€” this is domain rigor most plotting libs lack.

## 4. Architecture & design-choice critique

**Closed 26-skill enum vs a compositional grammar â€” `defensible`, and correct for
the threat model.** The LLM-eval literature (Â§3.2) shows agents struggle with open
composition algebras; a bounded enum is far easier to validate, route, and repair.
The cost is real â€” when an agent needs a figure not in the enum, it has *no path*
except the `scene:null` host escape hatch â€” but for a safety-first agent target this
is the right trade. **Recommendation:** keep the enum; add one Gosling-style
compositional "custom panel" skill later if demand appears, not now.

**Hand-written 2657-line registry + 2299-line params â€” `questionable`.** The four
matrix skills and two degree skills are copy-paste (`registry.ts:914-1131`); the
constraint-kind names are triplicated with partial test coverage; `describeSkill`
hand-clones what the freeze loop hand-enumerates. The repo's own non-negotiable is
"single sources of truth â€” don't hand-edit derived data," and this is the largest
violation of its own rule. **Alternative:** a small factory that generates the
matrix/degree families from shared descriptors (cuts ~400 LOC and the drift risk),
and derive the constraint-kinds array + a "every used kind has semantics"
exhaustiveness test from the semantics object keys. This is the highest-leverage
maintainability fix.

**zod-as-contract + hand-authored JSON-Schema manifest â€” `defensible` but
un-parity-tested.** `JSON_PARAMS_PORTABLE_SCHEMA` is `additionalProperties:true` and
expresses *none* of the runtime exact-JSON guarantees, so a non-TS host validating
only against the manifest gets a materially weaker gate than the TS runtime
(`vizSpec.ts:94-104`). No test asserts the hand-authored portable schema stays in
parity with the zod shape. **Fix:** either generate the portable schema from zod, or
add a parity test; and document that the manifest is a *description*, not an
*equivalent gate*.

**Exact-JSON gate + amplification-safe preflight ordering â€” `sound`.** Preflighting
before the heavy parse is the correct DoS guard, and it is done in both entrypoints.
The one caveat (Proxy TOCTOU on the raw pre-clone fuse) does not affect
*correctness*, only amplification-avoidance, and the 500k global cap still bounds it.

**Fail-closed honesty + self-asserted flags â€” `sound design, oversold claim`.** The
enforcement (literal-false pin, append-only caller note, bidi isolation) is
genuinely good. But (a) it is 100% load-bearing on one literal (finding 4), (b) the
"flags only, never caller text" claim is false because `mandatoryDisclosure` sniffs
`p.source` (finding, Â§2.5), and (c) it verifies *labels*, not *data*. **Fix the
docs first** (they currently overclaim), then consider gating the caption on a
positive evidence token rather than the absence of four negatives.

**`scene:null` as convention not enforcement â€” `defensible` (unavoidable for a pure
library).** A validation library *cannot* force an external matplotlib process to
render a caption. The honest framing is "Cortexel guarantees the caption is computed
and handed over; the host is contractually obligated to show it." That is fine â€” but
SECURITY.md's "must" should be softened to "the library cannot enforce this; hosts
that drop the caption are non-conformant," and a conformance test-kit for hosts
would give the obligation teeth.

**Committed `dist/` â€” `sound`.** Verified deterministic; the drift gate is a real
guard; it enables zero-build git installs. The cost (noisy diffs, every PR must
rebuild) is accepted knowingly. Keep it.

**The five design laws â€” `laws for 3 & 4, aesthetics for 1 & 2`.** Law 3 (honesty
fails closed) and law 4 (allocation-free `useFrame`) have executable guards and are
verifiable â€” and law 4 verifiably holds in the KG scene. Laws 1 & 2 (sphere/voxel
grammar, emissive â‰¤1.15) are *aesthetic conventions* dressed as laws; law 2 has a
guard but law 1 is unfalsifiable. Fine to keep, but call them "conventions" where
they are conventions.

**The 70 KB `ReferenceChartScene` â€” `defensible, at its ceiling`.** All pure geometry
is correctly factored into `chartGeometry.ts`/`topologyGeometry.ts`; the file is big
because 15 charts share a `ChartShell`. The maintenance smell is the ~30-line shell
prop block repeated 15Ã— and the absence of any `useMemo` (every chart rebuilds
domains/Sets/Maps per render). **Fix:** one file per skill under `charts/skills/`
with a generated dispatcher; memoize geometry by params identity.

**Version triple/quadruple â€” `questionable`.** Four independent version axes with no
unifying guard means no single number tells a reviewer whether the contract changed,
and the manifest version '8' is a hand-typed literal that the smoke test also pins to
'8' â€” so a manifest content change with a forgotten bump passes every gate. **Fix:**
derive `manifestVersion` from a content hash, and document the version matrix in one
place.

**Standalone-source governance â€” `resolved`.** This repository is the canonical,
writable Cortexel source. Engram is a downstream consumer that pins published commits
and carries generated contract snapshots; it must not recreate an in-tree Cortexel
source copy. CONTRIBUTING and the agent guide now state the same workflow.

## 6. The ten lenses

**1. Contract & agent DX â€” `strong loop, closed-enum ceiling`.** The
discoverâ†’authorâ†’validateâ†’repair loop is genuinely closable by a model with no human:
structured repair carries hint + example + `didYouMean`, and it is prompt-injection
safe. The ceiling is the closed enum â€” when the agent needs a figure not in the 26,
its only path is the `scene:null` host hatch. *Finding:* routing to a "weak" skill
drops the `weakDisclosure` signal on the success path (`router.ts:61-62`), so an
agent isn't told a derived-view disclosure is mandatory without a second call.

**2. Security & adversarial robustness â€” `best-in-class for the class, three real
leaks`.** The envelope hardening is the strongest part of the codebase and is novel
for a viz spec. But honesty *fails open* on finding 1, three paths leak hostile text
into agent-facing output (findings 2/3), and a TypedArray-subclass getter leak
breaches the no-accessor invariant. None are critical; all are fixable in a day.

**3. Scientific correctness & epistemics â€” `honest form, asserted substance`.**
Binning is correct (half-open, bounded snap, fail-closed). But provenance flags are
self-asserted, nothing checks data-vs-claim, there is **no uncertainty
representation**, and the analyses are an **unvalidated reimplementation of
Elephant**. The disclosure caption is the only defense against a confidently-flagged
fabrication â€” and it leans entirely on one literal pin (finding 4).

**4. Prior art, novelty & positioning â€” `two real contributions, one wrong word`.**
Novel and citable: fail-closed in-spec honesty; hostile-input hardening. Reinvention:
the repair loop (VegaChat/LIDA), the provenance object (worse than PROV), the
analyses (Elephant). Wrong word: "grammar." See Â§3.9.

**5. Renderer & graphics â€” `sound, with a DoS gap and a memoization gap`.** `useFrame`
is verifiably allocation-free; instancing, culling, bloom-budget, and CDN-free labels
are all handled correctly. Gaps: non-compacting charts have no budget (DoM DoS), no
`useMemo` anywhere in the chart scene, and headless export is absent (roadmap).

**6. Accessibility & perception â€” `right pattern, partial coverage`.** DOM companions
for WebGL and paginated mirrors match Data Navigator's philosophy; `crameri` default
is good. But only 5/15 SVG charts expose values, palette contrast/CVD is untested,
and the emissive-bloom neuron style is hostile to low-vision users with no documented
alternative.

**7. Interoperability & ecosystem fit â€” `the existential question`.** Cortexel is TS
in a Python field. NEST users live in PyNEST+Elephant+Viziphant+Neo+matplotlib. The
language-neutral manifest is a real bridge *only if a Python validator ships* (it
doesn't yet â€” it's aspirational, unlike MolViewSpec). Ranked unlocks: Python/Pydantic
validator > Elephant/Neo adapters > MCP server > headless export > Jupyter/anywidget.

**8. Architecture & maintainability â€” `disciplined, but violates its own single-source
rule`.** The axis wiring (SCENE_NAMES, SKILL_IDSâ†’registry, derived router) is
exemplary. The exceptions are large: 2657-line hand-written registry with copy-paste
families, triplicated constraint kinds, hand-mirrored clone/freeze loops, and
hand-authored portable schemas with no parity test â€” all drift risks tests don't
fully catch.

**9. Testing, verification & release â€” `broad shape coverage, zero value coverage`.**
441 tests, many as executable invariant guards (honesty, design laws, manifest
parity) â€” genuinely good. But **no property-based/fuzz tests** on the JSON gate or
transforms, and **no golden-value tests** against reference outputs. The version
quadruple is incoherent to a reviewer. `react-test-renderer` is deprecated and should
migrate to `@testing-library/react`.

**10. Product, adoption & governance â€” `strong artifact, invisible product`.** The
positioning is over-worded ("grammar"), there is no live gallery or playground (you
cannot see a figure without installing), and there is no published validation study
against real NEST data. Governance no longer blocks external code PRs: this standalone
repository is the canonical writable source, while Engram consumes pinned commits and
generated contract snapshots. The remaining sequenced steps in Â§5 are the path from
"impressive repo" to "tool neuroscientists adopt."
