# Cortexel 0.9.0 — adversarial review and reconciliation (2026-07-15)

An adversarial multi-lens review (science, security, honesty, contract-coherence) ran
against the 0.9.0 implementation. Each lens hunted for defects it could demonstrate with
a concrete input; every claimed defect was then independently verified with the goal of
REFUTING it. **12 findings survived verification** (2 P0, 2 P1, 5 P2, 3 P3). All 12 are
resolved below, each with a regression test where one was missing.

## P0 — honesty (both in the render disclosure path)

1. **`disclosureFacts` never populated the scope/topology/spatial facts.** A rank-local,
   sampled, or incomplete-universe network figure rendered with none of its mandatory
   disclosures, because the facts those rules fire on were never derived from the request.
   **Fixed:** `disclosureFacts` now derives `scopeKind`, `rank`, `worldSize`, sampled
   counts, `nodeUniverseComplete`, `multapseAggregation`, `schematicLayout` (from the
   layout mode), `positionsMissing`, and defaults `uncertaintyKind` to `none` so an
   omitted band still discloses its absence. Guarded by `test/disclosureCompleteness.test.ts`.

2. **The `forced` disclosures argument was hardcoded empty** (`['LAG_ORIENTATION'].filter(() => false)`),
   so every compiler-forced rule (`LAG_ORIENTATION`, `ZERO_LAG_SELF_PAIRS_EXCLUDED`,
   `ABSENT_IS_NOT_ZERO`, `SCHEMATIC_LAYOUT`) never fired. **Fixed:** a `forcedDisclosures`
   function returns the real per-skill forced set; a correlogram now states its lag
   orientation, a matrix states absent-is-not-zero, and the schematic graph states its
   layout is non-spatial. Guarded by the same test.

## P1 — science

3. **Matrix value array lost index alignment on a null.** `numbers()` filtered out a
   null/missing weight or delay before `computeMatrix`, shifting every later edge onto
   another cell's value. **Fixed:** the value array stays index-aligned and `computeMatrix`
   skips a null per-cell. Guarded by a hand vector in `test/analysis.test.ts`.

4. **A pre-binned PSTH rendered as an all-zero figure.** The PSTH branch only handled
   `mode: events`. **Fixed:** a `mode: prebinned` branch draws the supplied counts,
   index-aligned with the edges (a null bin renders empty, never filtered). It also honours
   the declared `normalization`. Guarded by a render test on the contract's own prebinned
   example.

## P2

5. **Binned branches hardcoded `finalEdgeInclusive: true`.** **Fixed:** the boundary is now
   read from the request's bin spec (`binBoundaryInclusive`) for the population-rate, ISI,
   distribution, and PSTH branches.

6. **`UNCERTAINTY_NOT_PROVIDED` fired only on an explicit `kind: none`.** A caller could
   suppress it by omitting the field. **Fixed:** an omitted uncertainty now fails closed to
   the same `none` fact, so the disclosure always fires when uncertainty is absent.

7–8. **Themes and budget profiles had two authorities** (a hand-copied enum in
   `common.v1.schema.json` and the registries). **Fixed:** the generator emits `themeId` and
   `budgetProfileId` enums into `registry-enums.v1.schema.json` and the presentation schema
   `$ref`s them — one authority, matching the existing unit/error/skill pattern.

9. **The generator's typed-keyword check omitted numeric keywords.** **Fixed:** `minimum`,
   `maximum`, `exclusiveMinimum`, `exclusiveMaximum`, and `multipleOf` are now covered.

## P3

10. **PSTH ignored the declared normalization** — resolved together with finding 4.

11. **`cortexel migrate` used raw `JSON.parse`,** bypassing the strict parser's limits and
   duplicate-key rejection on exactly the untrusted legacy input that most needs them.
   **Fixed:** it now routes through `parseJsonStrict`.

12. **The renderer was declared in both the skill contract and the capability record** with
   no drift check. **Fixed:** the generator now asserts the two agree.

## Outcome

All 12 confirmed findings resolved. Full suite after fixes: **921 tests passing across 36
files**; cross-language digest parity preserved; generated files fresh and deterministic.
No P0/P1/P2 finding remains open.
