## What the baseline tree contained

- A `VizSpec` envelope (`core/vizSpec.ts`) combining request fields, provenance
  flags, renderer route, and disclosure behavior in one caller-authored object.
- A closed skill registry of **26** entries (`core/skills/registry.ts`, 2,657
  lines) including deprecated aliases, four host-only `scene: null` routes, a 3D
  scene, and a force-directed knowledge graph.
- Genuinely strong hostile-input handling (`core/safeRuntime.ts`,
  `core/nest/safeInput.ts`): accessor-free snapshots, bounded diagnostics,
  intrinsic typed-array length reads, bidi/control-character display guards.
- NEST-oriented analysis and topology transforms (`core/nest/`).
- A React chart monolith (`react/charts/ReferenceChartScene.tsx`, 1,834 lines).
- A committed `dist/` (47 files) because Git-dependency installs had no build step.
- A generated `dist/skills.manifest.json`.

## Known limitations of the baseline (carried forward as work, not as claims)

These were identified by the v1.0 review and are the reason the tree was **not**
tagged `1.0.0`:

1. Package version, VizSpec version, skill-axis version, manifest version, and
   constraint-language version were separate uncoordinated axes.
2. Handwritten Zod schemas, portable JSON Schema, manifest constraints, adapter
   logic, and renderer assumptions could drift apart.
3. The catalog mixed canonical renderers, experiments, deprecated aliases, and
   host-only routes Cortexel could not enforce.
4. Scientific calculations were internally self-consistent but had **no**
   independent reference vectors from NEST, Elephant, or hand derivation.
5. Callers could influence fields that should be library-generated assurances.
6. Headless SVG export did not exist; rendering required React.
7. There was no Python implementation, so the portable schema had no independent
   second reader.
8. There was no immutable release, no published package, no provenance, and no
   citation metadata.

## Review limitation inherited from the blueprint

The v1.0 review that produced the blueprint could not resolve `github.com` from
its shell. It therefore never cloned this repository and never ran its tests. The
test results in this document are the **first** independent execution and are
recorded as receipts, not as inherited claims.
